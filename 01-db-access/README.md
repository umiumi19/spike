# スパイク1: DBアクセス書き比べ(ent / go-jet / sqlc)★本命

## 全ツール共通の「お題」
同じ複雑クエリを3通りで書き、書き心地を比べる。

> **指定した userId のユーザーを、その「削除されていない投稿」と、各投稿の「削除されていないコメント」まで取得する。**
> (3テーブルの関係/JOIN + 論理削除フィルタ + 入れ子で結果を持つ)

これを書いてみると、各ツールの「関係をどう表現するか」「入れ子結果をどう組み立てるか」の差が一番はっきり出る。

## 観察ポイント
- 関係(posts→comments)の表現がどれだけ自然か
- 入れ子(User→[]Post→[]Comment)へのマッピングが自動か手作業か
- 削除フィルタ(deleted_at IS NULL)の差し込みやすさ
- 動的条件(例: タイトル部分一致を任意で足す)を入れたときの書きやすさ
- 生成コード/ビルドエラーで型ミスを捕まえられるか

## 各ツールのセットアップは各サブフォルダの README へ
- `ent/`   … 採用候補(本命)
- `jet/`   … drizzle に最も近い対抗
- `sqlc/`  … 元の案(比較用)

注: 3ツールはスキーマとの関係が違う(これ自体が比較ポイント)。
- ent: スキーマを **Goで定義** → コード生成(DBは ent が作れる)
- jet/sqlc: **既存DBスキーマ**(`../schema/schema.sql` を適用したもの)から生成
